---
description: Clean Code
---

Contexto: sempre prezar por componentes menores, dividindo componentes grandes com + de 200 linhas em componentes menores que se aclopam no principal (Pai/Mãe)

REGRAS:

1. Nunca criar componentes acima de 200 linhas de código
2. Usar operadores ternários para diminuir linhas de código em situações de lógica reduzindo cadeias de ifs e elses
3. Aplicar reutilização de código/componentes que podem ser reaproveitados
4. criar/ajustar sub-componentes menores seguindo a regra 1 de não ter componentes gigantes com mais de 200 linhas de código
5. Comentar com estrategias de documentar cada componente , auto-explicando partes importantes do código
6. Escrever código limpo dividindo bem, lado do servidor e lado do cliente.
